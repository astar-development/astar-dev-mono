# Git workflow

This repo uses a trunk-based approach - `main` MUST always be deployable.

All new work MUST be created on a feature branch.

Feature branches should follow the below naming convention:

- feature/short-description-of-the-feature
- bug/short-description-of-the-bug-with-issue-id
- doc/short-description-of-the-docs-being-added-or-updated
